How to use this deck

For students (self-study)

  1. Pick a topic with the buttons below, or stay on All topics.
  2. Read the front, say the definition out loud, then click the card to flip and check yourself.
  3. Hit ✓ Got it to retire a card you know, or ↻ Review again to keep it in rotation and move on.
  4. Use 🔀 Shuffle so you don't memorize the order, and Reset deck to start the topic over.
  5. When the deck empties ("Deck complete!"), you've mastered that topic — switch topics and repeat.

Tip: click the card to focus it, then use ← / → to move between cards.

Want these cards in your own study page?

The whole deck is just JSON consumed by the shared TG.flashcards engine. To build your own mini-deck, drop a script block and a mount point on any page that links assets/app.js:

HTML 
<!-- 1) The data: an array of { front, back, tag? } -->
<script type="application/json" id="my-cards">
[
  { "front": "Composable function",
    "back": "A Kotlin function annotated with @Composable that describes a piece of UI.",
    "tag": "Compose UI" },
  { "front": "remember { }",
    "back": "Caches a value across recompositions so it is not recreated every frame.",
    "tag": "State" }
]
</script>

<!-- 2) The mount point: the engine auto-inits on this attribute -->
<div data-flashcards="my-cards"></div>

Or mount it manually from your own JavaScript (this is exactly what the topic-filter buttons on this page do):

JavaScript 
// Filter the full deck to one tag, then (re)mount into #deck-mount.
const all = JSON.parse(document.getElementById("all-cards").textContent);
const stateCards = all.filter(c => c.tag === "State");

// TG.flashcards(mountSelectorOrEl, cardsArrayOrScriptId)
TG.flashcards("#deck-mount", stateCards);
