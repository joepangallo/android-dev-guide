Format & prerequisite

3.0 credit hours, face-to-face. We meet 3 days a week for 4 hours across roughly 4 weeks (each syllabus "Week" = 3 of our sessions). Prerequisite is COP1411C Data Structures & Algorithms: students already program and know OOP (Java/C++/Python), but Kotlin is new — so it is front-loaded into Session 2. There is no textbook; we use the official Android Basics with Compose course and Android Developer docs.

Concept tools (interactive labs)

6 hands-on explorers — run them live in class or assign as practice

🔄 Activity Lifecycle Explorer Drive an Activity through onCreate → onStart → onResume → onPause → onStop → onDestroy and watch callbacks fire on rotation and backgrounding. Pairs with Session 3.
🧭 Explicit vs Implicit Intents Explorer Build intents, attach extras, and see which one launches a named Activity versus which one asks Android to resolve an action (dial, view URL, share). Pairs with Session 3.
🎨 Compose Layout & State Playground Tweak Column/Row/Box arrangement, modifiers, and a remember state value, and watch recomposition update the preview. Pairs with Sessions 4–5.
📇 Kotlin for Android — Interactive Cheat Sheet Searchable reference for val/var, null-safety (?., ?:), data classes, lambdas, and scope functions — with Java-to-Kotlin comparisons. Pairs with Session 2.
🗃️ SQLite / Room CRUD Simulator Run Create / Read / Update / Delete against a mock table and see the generated Room @Dao queries and resulting rows. Pairs with Session 9.
📡 Networking Pipeline Visualizer Trace a request end-to-end: HTTP request → JSON response → parse → UI state, the Retrofit + serialization path used in the final project. Pairs with Sessions 10–11.

Master it — your practice instrument

Self-directed drills that track your mastery, not a grade

▶
To run code live, start the local server

The auto-checked challenges compile on JetBrains’ Kotlin server, which only accepts a real http origin — so the Run buttons can’t reach it when you open these pages straight from a file. Double-click serve.command in the teaching-guide folder (it opens the guide at http://localhost:8765) and Run works everywhere. Opened as a plain file, every snippet still shows its code with a one-click Open in Kotlin Playground.

🚀
How to become the strongest engineer in your cohort

Reading the sessions is the warm-up. You actually get good by doing four things on repeat — and this instrument is built for all four:

1 · Deliberate practice. Don't re-read code — write it. Open a challenge, edit the // YOUR CODE region, and hit Run. You've got it when every case goes green. Each pass nudges your mastery up.

2 · Retrieval, not recognition. Pull the answer out of your own head before you peek. Spaced-repetition cards and the predict-then-run prompts force recall — the single highest-yield study move there is.

3 · Spacing. Five focused reps today plus five tomorrow beats fifty in one sitting. Come back to your due reviews daily; the deck schedules each card for the day you're about to forget it, and keeps your streak alive.

4 · Build real things. Skills stick when they ship. Climb the project ladder rung by rung until you can assemble the whole app — domain model, repository, UI state — from memory.
