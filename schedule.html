🛠️ The Course Project

Required feature checklist

Build an app of your choice. To call it complete, it should include all of the following — the same bar a senior engineer would expect in a code review.

  • Two or more screens with navigation — Navigation-Compose (NavHost, routes, arguments).
  • A Jetpack Compose UI driven by state — state hoisting; the UI is a function of state.
  • Local persistence — Preferences DataStore and/or Room, so data survives an app restart.
  • Data from a web API over the network — Retrofit + coroutines + JSON, with loading / error / empty states.
  • Clean architecture — ViewModel + repository, unidirectional data flow.

🎯 Course Learning Outcomes & Where They're Taught

Upon completion of COP4667 you will be able to:

CLO 1 Android ecosystem & essentials

Understand the essentials of Android development and its ecosystem.

Taught in S1, S3, S8, reviewed S12.

CLO 2 Designing UIs & wiring them to code

Design user interfaces using components and UI elements and connect them to code.

Taught in S4, S5, S6, S7, S8.

CLO 3 Building apps in Kotlin

Create various types of Android applications using an industry-current language (Kotlin).

Taught in S1, S2, S3, S5.

CLO 4 Local data storage & databases

Explore techniques for application data storage using local storage and/or a database.

Taught in S7, S9, S11.

CLO 5 Integrating APIs

Integrate APIs into Android applications (location services, cloud-based storage).

Taught in S10, S11.

CLO 6 Network protocols for backend comms

Understand and implement different network protocols for backend communication.

Taught in S10, S11.

Tools & expectations

Bring a laptop that can run the latest Android Studio plus the Android Emulator (or a physical device with USB debugging). We code in Kotlin with Jetpack Compose and Material 3. There is no textbook — all readings are the official, free docs at developer.android.com. Complete the listed reading before each session so class time can focus on live coding and labs.
