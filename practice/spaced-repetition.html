Why spacing makes knowledge permanent

Forgetting is the feature, not the bug

The moment after you learn something, your memory of it starts to fade — that's the forgetting curve. Cramming fights it by re-reading the same thing ten times in one night; the trace decays anyway and is gone by the exam.

Spaced repetition does the opposite. It lets the memory start to fade, then asks you to recall it right at the edge of forgetting. Each successful recall makes the trace decay more slowly, so the next review can wait longer. The result: with fewer total minutes, the knowledge lasts for months instead of hours.

Two effects working together

  • The spacing effect. Reviews spread out over days beat the same number of reviews packed into one sitting. Distance in time is what builds durable memory.
  • The testing effect (active recall). Pulling an answer out of your own head strengthens memory far more than re-reading it. That's why every card hides the answer until you've tried — the struggle is the learning.

This trainer combines both: it tests you (recall), and it schedules each card at a growing interval (spacing).

How to use it every day

The daily habit (3 steps)

  1. Open it once a day — same time if you can (right after class, or first thing each morning). Consistency beats marathon sessions.
  2. Clear the queue. Read the front, say the answer out loud or in your head before revealing, then click Show answer and grade yourself honestly.
  3. Stop when it says "All caught up." That's the schedule protecting your time — there's nothing useful to review yet. Come back tomorrow and the next batch will be waiting.

Keyboard: Space or Enter reveals the answer; then 1 Again · 2 Hard · 3 Good · 4 Easy.

How to grade yourself honestly

  • AgainYou blanked or got it wrong. The card resets and comes back this session. Be honest here — "Again" is where real learning happens.
  • HardYou got it, but it was a struggle or took a while. Shorter next interval.
  • GoodYou recalled it correctly with normal effort. The default — use it most.
  • EasyInstant and effortless. Longer next interval so you don't waste time on it.

Grading "Good" on something you actually fumbled only cheats future-you. The algorithm is only as smart as your honesty.

Want this deck on your own page?

The whole deck is just JSON consumed by the shared MASTERY.srs engine. Drop a script block and a mount point on any page that links assets/mastery.css + assets/mastery.js:

HTML 
<!-- 1) The data: { id, cards: [ { id, topic, front, back } ] } -->
<script type="application/json" id="deck">
{
  "id": "my-deck",
  "cards": [
    { "id": "k1", "topic": "kotlin",
      "front": "val vs var?",
      "back": "<strong>val</strong> = read-only; <strong>var</strong> = reassignable. Prefer val — fewer moving parts, fewer bugs." }
  ]
}
</script>

<!-- 2) The mount point -->
<div id="srs"></div>

<!-- 3) Mount it (the engine reads the schedule from this device) -->
<script>MASTERY.srs("#srs", "#deck");</script>
ℹ️
No build step

Everything here is hand-written HTML/CSS/vanilla JS and works by double-clicking the file (file://) — no CDN, no framework, no internet required. Each card needs a topic so the scheduler can track maturity per topic and feed your personal mastery.
